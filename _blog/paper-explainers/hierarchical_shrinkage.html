Hierarchical Shrinkage

Decision trees tend to overfit. Hierarchical shrinkage reduces that by adjusting node predictions after the tree is grown, with a single parameter.

Agarwal, Tan, Ronen, Singh, Yu  ·  Paper (ICML 2022)
scroll to begin

BackgroundA tree predicts with local averages

A decision tree splits the data into smaller and smaller groups, then predicts the average outcome within each final group, or leaf.

How well it fits depends on how far it splits. The figure below shows a tree on the left and the prediction it makes on the right. As you scroll, the tree grows one level at a time, and the prediction changes with it.

No splits

Start with the average

A tree with no splits predicts one value for every input: the overall mean. On the right that is a flat line. It captures none of the structure in the data.

One split

The first division

The first split separates the data into two groups, giving two levels. The prediction starts to bend toward the shape of the data.

A few splits

Following the shape

With a few more splits the staircase begins to track the three levels in the underlying signal. Each leaf on the left is one step on the right, drawn in the same color.

About right

Enough structure

Around here the steps line up with the three levels. The tree has enough structure to describe the signal without much excess.

Going deeper

Leaves get small

Deeper splits cut the data into smaller groups. The steps multiply, and the prediction starts to react to individual points.

Too big

The tree overfits

At full depth the prediction is a jagged staircase. The smallest leaves hold only a few points, so their averages follow noise rather than signal.

The rest of the page is about pulling those small leaves back.

The reframeA prediction is a path through the tree

To reduce that overfitting without changing the splits, it helps to view a prediction not as one number from a leaf, but as a path from the root down to that leaf.

The next figure uses a smaller tree so the parts stay readable. The left panel is the structure and the right panel is the staircase it predicts across x. Each leaf produces one step, drawn in the same color so the two views line up.

Two views, one tree

Structure on the left, prediction on the right

This is a smaller tree so the parts stay readable. Each leaf maps to one step of the staircase, and matching leaf and step share a color.

Means everywhere

Every node stores a mean

Not only the leaves. The root holds the average over all the data; each child holds the average over its subset. Circle size shows how many samples a node contains.

Following a path

Root to leaf

To predict for a point, walk the highlighted path down to a leaf. A standard tree reports that leaf's mean and ignores the nodes above it. The highlighted leaf is the highlighted step.

Rewriting the value

The leaf value as a sum of changes

Write it as the root mean plus the change in mean at each step down the path. These changes sum back to the leaf value, so the prediction is unchanged for now.

The adjustment

Divide each change by 1 + λ/N

Here N is the sample count of the parent node. On the tree, an arrow draws every node toward its parent, not just the leaves; its length is the fraction of the gap closed. The matching arrows on the right show the leaves' motion: each step slides from its raw mean (dashed) toward its shrunk value.

Arrows are colored to match each node. Nodes with small N close more of the gap, so their arrows are longer, which is why the deepest leaves move most.

Turning the dial

Larger λ, flatter predictions

As λ grows the arrows lengthen: step heights move toward the overall mean (dashed line) and node values move toward the root. The tree's structure does not change; only the predicted values do. At λ = 0 you recover the original tree; as λ grows large every prediction approaches the global average.

The trade-offHow λ trades bias against variance

λ sets how strongly predictions are pulled toward their parents. The top plot is the fit; the bottom plot is the error on a held-out test set as λ changes. Scroll to move along the curve.

Small λ keeps the overfit staircase. Large λ flattens the model too much. The lowest test error sits in between, which is the value HSTreeCV selects by cross-validation.

λ = 0

No shrinkage

The tree is unchanged: the fit is the jagged staircase. The dot sits at the left of the curve, where test error is high because the model is overfit.

Small λ

The staircase settles

A little shrinkage pulls the noisiest steps toward their parents. The fit smooths out and test error starts to fall.

Lowest test error

The balance point

At the marked λ the fit follows the three levels closely and test error reaches its minimum. This is the value cross-validation aims for.

Large λ

Over-flattened

Push λ further and predictions collapse toward the overall mean. The fit underfits, and test error climbs again on the right of the curve.

PropertiesWhat the method does and does not change

Hierarchical shrinkage does not alter the tree's structure. It only rescales the changes in predicted value along each path, which makes it fast to apply and easy to add to an existing tree pipeline.

It improves predictive accuracy for individual trees, and since it works on one tree at a time, it applies to each tree in a Random Forest or other ensemble.

1

One parameter

A single λ controls all of the regularization, usually chosen by cross-validation.

post-hoc

No retraining

Applied after the tree is grown. The structure stays fixed; only the predicted values move.

per-tree

Trees and ensembles

Works on CART, Random Forest, ExtraTrees, and gradient boosting by shrinking each tree.

UsageHow it looks in code

In imodels it is used like any scikit-learn estimator:

from imodels import HSTreeClassifierCV, get_clean_dataset
from sklearn.model_selection import train_test_split

X, y, feat_names = get_clean_dataset('csi_pecarn_pred')
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.33)

# lambda is selected by cross-validation
model = HSTreeClassifierCV(max_leaf_nodes=7)
model.fit(X_train, y_train)
preds = model.predict_proba(X_test)

# also works on ensembles: pass the estimator
from sklearn.ensemble import RandomForestClassifier
from imodels import HSTreeClassifier
model = HSTreeClassifier(estimator_=RandomForestClassifier())
